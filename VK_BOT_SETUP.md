# Настройка VK бота для системы бронирования

## Обзор

Этот документ описывает процесс настройки VK бота для интеграции с системой бронирования услуг красоты.

## Предварительные требования

1. Аккаунт ВКонтакте
2. Созданная группа ВКонтакте
3. Доступ к настройкам группы

## Шаг 1: Создание VK приложения

1. Перейдите на [vk.com/dev](https://vk.com/dev)
2. Нажмите "Создать приложение"
3. Выберите тип "Standalone приложение"
4. Заполните название и описание
5. Получите **App ID** и **Secure key**

## Шаг 2: Настройка группы ВКонтакте

### 2.1 Создание группы (если еще не создана)

1. Перейдите в раздел "Группы" → "Создать группу"
2. Выберите тип "Бизнес" или "Тематическая"
3. Заполните информацию о группе
4. Получите **ID группы** (из URL: vk.com/club**123456**)

### 2.2 Получение токена группы

1. Перейдите в "Управление" → "Настройки" → "Работа с API"
2. Создайте ключ доступа с правами:
   - Доступ к сообщениям сообщества
   - Доступ к управлению сообществом
3. Скопируйте **токен доступа группы**

### 2.3 Настройка Callback API

1. В разделе "Работа с API" выберите "Callback API"
2. Укажите URL сервера: `https://ваш-домен.ru/api/vk-bot/webhook`
3. Укажите секретный ключ (любая строка)
4. Выберите версию API: **5.131**
5. Включите события:
   - Новое сообщение (`message_new`)
   - Ответ на сообщение (`message_reply`)

### 2.4 Настройка сообщений

1. Перейдите в "Сообщения" → "Настройки для бота"
2. Включите "Возможности ботов"
3. Настройте приветственное сообщение (опционально)
4. Включите клавиатуру бота

## Шаг 3: Конфигурация переменных окружения

Добавьте следующие переменные в ваш `.env` файл:

```env
# VK Notifications (для уведомлений админов)
VK_ACCESS_TOKEN="ваш-токен-группы"
VK_GROUP_ID="123456"
VK_ADMIN_IDS="admin_id1,admin_id2"

# VK Bot (для работы бота)
VK_BOT_ACCESS_TOKEN="ваш-токен-группы"
VK_CONFIRMATION_TOKEN="ваш-секретный-ключ"

# Безопасность cron задач (опционально)
CRON_SECRET_TOKEN="случайная-строка-для-безопасности"

# Базовый URL для API вызовов
NEXT_PUBLIC_BASE_URL="https://ваш-домен.ru"
```

### Описание переменных:

- **VK_ACCESS_TOKEN** - токен для отправки уведомлений админам
- **VK_GROUP_ID** - ID вашей группы ВКонтакте
- **VK_ADMIN_IDS** - ID админов через запятую (получить можно на [vk.com/dev/users.get](https://vk.com/dev/users.get))
- **VK_BOT_ACCESS_TOKEN** - токен бота (обычно тот же, что и VK_ACCESS_TOKEN)
- **VK_CONFIRMATION_TOKEN** - секретный ключ для подтверждения webhook'ов
- **CRON_SECRET_TOKEN** - токен для защиты cron endpoints
- **NEXT_PUBLIC_BASE_URL** - базовый URL вашего сайта

## Шаг 4: Подтверждение сервера

1. Запустите ваше приложение
2. В настройках Callback API нажмите "Подтвердить"
3. Сервер должен ответить строкой подтверждения
4. После успешного подтверждения включите Callback API

## Шаг 5: Тестирование бота

1. Найдите вашу группу ВКонтакте
2. Напишите сообщение группе
3. Отправьте команду "Начать" или "Привет"
4. Бот должен ответить приветственным сообщением

### Доступные команды:

- **Начать** / **Привет** - приветствие и главное меню
- **Прайс** / **Прайс-лист** - показать услуги и цены
- **Записаться** - начать процесс бронирования
- **Помощь** - справочная информация
- **Отмена** - отменить текущее действие

## Шаг 6: Настройка напоминаний (опционально)

Для автоматических напоминаний настройте cron job:

```bash
# Каждый час проверять напоминания
0 * * * * curl -X POST -H "Authorization: Bearer ваш-cron-токен" https://ваш-домен.ru/api/vk-bot/reminders
```

Или используйте сервисы типа [cron-job.org](https://cron-job.org) для внешних cron задач.

## Шаг 7: Мониторинг и логи

### Проверка статуса бота:

```bash
curl https://ваш-домен.ru/api/vk-bot/webhook
```

### Проверка предстоящих напоминаний:

```bash
curl https://ваш-домен.ru/api/vk-bot/reminders
```

### Логи в консоли:

Бот выводит подробные логи в консоль сервера:
- Входящие сообщения
- Обработанные команды
- Отправленные уведомления
- Ошибки API

## Устранение неполадок

### Бот не отвечает на сообщения

1. Проверьте правильность URL webhook'а
2. Убедитесь что сервер доступен извне
3. Проверьте токены в .env файле
4. Посмотрите логи сервера на ошибки

### Ошибки VK API

1. Проверьте права доступа токена
2. Убедитесь что версия API корректная (5.131)
3. Проверьте что группа не заблокирована

### Webhook не подтверждается

1. Проверьте что CONFIRMATION_TOKEN правильный
2. Убедитесь что сервер возвращает строку подтверждения
3. Проверьте доступность endpoint'а /api/vk-bot/webhook

### Напоминания не отправляются

1. Проверьте настройку cron job'а
2. Убедитесь что CRON_SECRET_TOKEN правильный
3. Проверьте логи endpoint'а /api/vk-bot/reminders

## Безопасность

1. **Никогда не публикуйте токены** в открытом коде
2. Используйте **HTTPS** для webhook'ов
3. Настройте **CRON_SECRET_TOKEN** для защиты cron endpoints
4. Регулярно **обновляйте токены** доступа
5. Мониторьте **логи на подозрительную активность**

## Дополнительные возможности

### Кастомизация сообщений

Отредактируйте файл `lib/vk-bot/message-formatter.ts` для изменения:
- Текстов сообщений
- Эмодзи и форматирования
- Клавиатур и кнопок

### Добавление новых команд

1. Добавьте новый тип команды в `lib/vk-bot/types.ts`
2. Обновите парсер в `lib/vk-bot/command-parser.ts`
3. Добавьте обработчик в `lib/vk-bot/command-handlers.ts`

### Интеграция с другими сервисами

Бот легко интегрируется с:
- Системами аналитики
- CRM системами
- Платежными системами
- Системами уведомлений

## Поддержка

При возникновении проблем:

1. Проверьте документацию VK API: [vk.com/dev/api_requests](https://vk.com/dev/api_requests)
2. Посмотрите логи сервера
3. Проверьте статус VK API: [vk.com/dev/health](https://vk.com/dev/health)
4. Обратитесь к разработчику системы
