# 🔐 Обновление системы безопасности

Система авторизации была обновлена для повышения безопасности.

## Что изменилось

### ✅ Улучшения безопасности

1. **Хеширование паролей** - Пароли теперь хешируются с использованием bcrypt
2. **JWT токены** - Вместо простых строк используются криптографически защищенные JWT токены
3. **Улучшенная валидация** - Проверка токенов в middleware с автоматическим удалением невалидных
4. **HttpOnly cookies** - Токены хранятся в защищенных cookies с флагом httpOnly
5. **Поддержка email** - Можно входить как по username, так и по email

### 🔧 Технические изменения

- Добавлены поля `email`, `createdAt`, `updatedAt` в модель Admin
- Создан модуль `lib/auth.ts` с функциями для работы с паролями и токенами
- Обновлен API роут `/api/admin/login` с проверкой хешированных паролей
- Обновлен `middleware.ts` с верификацией JWT токенов
- Добавлена переменная окружения `JWT_SECRET`

## Как обновить пароль администратора

### Вариант 1: Через Prisma Studio (рекомендуется)

1. Сгенерируйте хеш вашего пароля:
   ```bash
   npm run hash-password "ваш_новый_пароль"
   ```

2. Скопируйте полученный хеш

3. Откройте Prisma Studio:
   ```bash
   npm run db:studio
   ```

4. Перейдите в таблицу `Admin`

5. Найдите вашего администратора и вставьте хеш в поле `password`

6. Сохраните изменения

### Вариант 2: Через SQL

```sql
-- Замените YOUR_HASH на хеш из команды hash-password
UPDATE Admin SET password = 'YOUR_HASH' WHERE username = 'admin';
```

## Настройка JWT Secret

⚠️ **ВАЖНО для продакшена!**

Обязательно измените `JWT_SECRET` в файле `.env`:

```bash
# Сгенерируйте случайный ключ
node -e "console.log(require('crypto').randomBytes(32).toString('hex'))"

# Добавьте в .env
JWT_SECRET="ваш_сгенерированный_ключ"
```

## Проверка работы

1. Обновите пароль одним из способов выше
2. Перейдите на `/admin/login`
3. Войдите с новым паролем
4. Проверьте, что редирект на `/admin` работает корректно

## Безопасность токенов

- Токены действительны 7 дней
- Токены автоматически проверяются при каждом запросе к `/admin/*`
- При истечении токена происходит автоматический редирект на страницу входа
- Токены хранятся в httpOnly cookies (недоступны для JavaScript)

## Troubleshooting

### Не могу войти после обновления

1. Убедитесь, что пароль в БД хеширован (начинается с `$2a$`, `$2b$` или `$2y$`)
2. Проверьте, что `JWT_SECRET` установлен в `.env`
3. Очистите cookies браузера для вашего сайта
4. Попробуйте сгенерировать новый хеш пароля

### Ошибка "Invalid token"

1. Очистите cookies браузера
2. Проверьте, что `JWT_SECRET` не изменился
3. Попробуйте войти заново

### Не работает в продакшене

1. Убедитесь, что установлены зависимости: `bcryptjs`, `jsonwebtoken`
2. Проверьте, что `JWT_SECRET` установлен в переменных окружения
3. Убедитесь, что миграция базы данных применена
